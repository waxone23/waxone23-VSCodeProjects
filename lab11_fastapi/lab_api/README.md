# Lab API (Parts 3 and 4)

This folder contains a complete FastAPI solution for:
- Part 3 - FastAPI basics
- Part 4 - Pydantic models and complete CRUD endpoints

## Files
- `main.py` - FastAPI app and endpoints
- `models.py` - Pydantic request/response models

## Run
1. Open a terminal in this folder.
2. Install dependencies:
   - `pip install -r requirements.txt`
3. Start server:
   - `uvicorn main:app --reload`
4. Open:
   - `http://127.0.0.1:8000/`
   - `http://127.0.0.1:8000/docs`

## Implemented Endpoints
- `GET /` -> `{ "message": "API is running" }`
- `GET /health` -> `{ "status": "ok" }`
- `POST /books` -> create book, returns `201 Created`
- `GET /books` -> list all books
- `GET /books/{book_id}` -> get one book, returns `404` if missing
- `DELETE /books/{book_id}` -> delete one book, returns `204 No Content` if deleted, `404` if missing

## Notes for assignment questions
- `/docs` is Swagger UI generated by FastAPI.
- `GET /books/hello` returns `422` because `book_id` must be an integer.
- `BookCreate` and `BookResponse` are separate because clients should not send server-generated `id` in create requests.
